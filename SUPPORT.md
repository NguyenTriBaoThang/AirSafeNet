# Support

## 1. Purpose

This document explains how to get help with AirSafeNet.

---

## 2. Where to Start

Before asking for help, please check:

- README.md
- ARCHITECTURE.md
- DEMO_SCRIPT.md
- MODEL_VERSIONING.md
- existing issues in the repository

---

## 3. Common Support Topics

Typical support requests may include:

- project setup issues
- AI server startup problems
- missing model file errors
- API request/response questions
- dashboard integration issues
- documentation clarification

---

## 4. How to Ask for Help

When opening an issue or requesting support, include:

- what you were trying to do
- what happened instead
- exact error message if any
- steps to reproduce
- operating system / runtime details
- screenshot or logs if useful

Good support requests are specific and reproducible.

---

## 5. Example Support Request

```text
Title: AI server fails to start after installing dependencies

Environment:
- Windows 11
- Python 3.11

Problem:
- Running `uvicorn app.api:app --reload --port 8000` throws ModuleNotFoundError.

What I tried:
- installed requirements.txt
- checked folder structure
- verified model.pkl exists
```

---

## 6. Best-Effort Support Scope

Because AirSafeNet is a student innovation project, support is best-effort.

Priority is usually given to:

- setup blockers
- demo blockers
- documentation gaps
- critical bugs affecting the main prototype flow

---

## 7. Recommended Channels

Suggested channels depending on repository setup:

- GitHub Issues for bugs and setup problems
- GitHub Discussions for broader questions
- project documentation for self-service help

---

## 8. Response Expectations

Response time may vary depending on project activity, academic workload, and competition deadlines.

Please keep questions concise and respectful.
